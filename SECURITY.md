# Security Policy

## Supported Versions

Only the latest version of Openpot is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| v0.8.x  | :white_check_mark: |
| < 0.8   | :x:                |

## Reporting a Vulnerability

We take the security of Openpot seriously. If you believe you have found a security vulnerability, please do NOT report it via a public GitHub issue.

Instead, please send a detailed report to **security@openpot.co**.

We will acknowledge receipt of your report within 48 hours and provide a timeline for addressing the issue. Once a fix is prepared and verified, we will release a new version and credit you for the discovery (if desired).

## Privacy Commitment

Openpot is a local-first application. We do not collect your session data. Security reports regarding data leakage or encryption weaknesses in our IndexedDB implementation are our highest priority.
